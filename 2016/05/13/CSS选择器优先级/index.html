CSS选择器优先级

  • CSS

浏览器通过判断优先级来决定哪些属性值与元素最相关，从而应用到该元素上。通常我们将CSS的优先级分为6级(由高到低)：

1.位于食物链最顶端的第一顺位优先的属性只要在属性后加上!import，它会覆盖页面任何位置定义的元素样式。IE6不支持这个属性，也可用作为一种hack。
2.第二顺位的优先属性是在html元素标签的style属性中定义的属性。但该方法会造成CSS难以管理，尽可能不使用。
3.第三顺位优先的属性是在id选择器中定义的属性。比如#id{margin:0;}会覆盖.class{margin:20px;}。
4.第四顺位的属性是在class选择器，属性选择器(比如a[href]{color:#eee;})，伪类选择器(比如a:hover{color:#ff0000;})中定义的属性。比如.class{margin:20px;}会覆盖tag{margin:40px;}。
5.第五顺位的属性实在元素选择器中定义的属性。比如tag{margin:40px;}会覆盖{margin:60px;}}。
6.最低一级的选择器为通配选择器，用{margin:60px;}}这样的类型来定义。

用一句话来描述CSS选择器优先级：当一个CSS选择器拥有更多高级别的属性时，它的优先级越高。比如#ID .class1 .class2{}的优先级会高于#ID .class2 div p span em，这是由于前者拥有1个三级和2个四级的选择器，而后者只拥有1个三级和1个四级(之后的五级标签更不用来参与比较)，所有前者优先级较高。如下(test颜色为#eee)：

CSS选择器是一个非常灵活的CSS属性，优雅的使用会使你的页面代码不再臃肿（反之结果往往会让你抓狂)，而且还可以作为hack的妙用之一。
当较多优先级相等的CSS选择器存在比较时，选择器定义的位置将决定一切(由高到低)：

1.定义于style标签内的CSS拥有最高的优先权。
2.位于style标签内的@import引入样式表(@import url(my.css))。
3.由link标签引入的样式表。
4.由link标签引入的样式表内的@import引入样式表。

多说一句：link引入的CSS在页面加载过程中同时被加载，而@import引入的CSS会等到页面被加载完再加载。
理解了这些CSS优先级问题后，也就不必苦恼于样式定于中的此类问题了
